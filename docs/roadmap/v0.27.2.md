# v0.27.2 — DataFusion Completeness, Code Hardening & Security

> Hardened the codebase against Assessment 1 and Assessment 2 findings.
> Completed DataFusion integration with auto-resolved `data_root`.
> Removed the `slateduck-sqlite-vfs` placeholder crate.

## Key Changes

- `SlateDuckCatalogProvider::from_catalog_store()` — resolves `data_root`
  automatically from catalog metadata; no second store open required (N-02)
- DataFusion async bridge replaced per-call thread spawn with a persistent
  background thread (N-05)
- All `DataRowEncoder` `.unwrap()` calls replaced with `.expect()` carrying
  diagnostic messages (N-03)
- `try_into().unwrap()` in keys.rs and values.rs replaced with
  length-guarded `.expect()` (N-07/N-08)
- `SocketAddr` construction uses `SocketAddr::from` instead of `parse().unwrap()` (N-12)
- Rowid arithmetic uses `checked_add` with `CatalogError::RowIdOverflow` (High-7)
- `SqlState` codes forwarded correctly from `SlateDuckError::SqlState` (High-9)
- `rebuild_catalog()` exists and is tested (F-10)
- `slateduck-sqlite-vfs` placeholder removed from workspace (N-06)
