# v0.27.1 — CDC Completeness & Real Parquet Row Scanning

> Completed the CDC (`table_changes()`) implementation. Before v0.27.1,
> CDC returned empty `columns_json`. After v0.27.1, `table_changes()` reads
> the actual Parquet files from the object store and returns real row data.

## Key Changes

- `extract_rows_from_parquet()` now opens actual Parquet files via `object_store`
- Row IDs assigned sequentially from `base_rowid` within each file
- `record_count` mismatch detection added (N-04): emits `slateduck_cdc_record_count_mismatch_total` metric when scanned rows differ from catalog metadata
- SQLSTATE `58030` returned on storage errors from `table_changes()` (not a panic)
